# 🎵 Track 5: Expensive Pain

> "Mistakes aren't failures. They're features you haven't fixed yet."

## What You're About To Do

Learn how to find and fix problems in your code. Everyone breaks things. The skill is knowing how to unbreak them.

## Bugs Happen

A bug is when your code doesn't do what you expected. Every developer deals with this daily. Here's how to handle it:

### Read the Error Message
Error messages look scary but they're actually directions. They tell you:
- **What** went wrong
- **Where** it happened (file name + line number)
- **Why** (sometimes)

### Use GitHub Issues
When you find a bug or want to track something to fix:

1. Go to your repo
2. Click the **Issues** tab
3. Click **New issue**
4. Describe the problem
5. Assign it to yourself

Issues are like a to-do list for your code. Professional developers use them every day.

### Ask Copilot
Stuck? Paste the error message into [Copilot Chat](https://github.com/copilot) and ask:
> "What does this error mean and how do I fix it?"

No shame in asking. Every senior developer does this.
