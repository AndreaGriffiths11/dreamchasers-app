## What You're About To Do

Learn how code gets saved and tracked. Every change you make is a **commit** — like a save point in a game. You can always go back.

## The Moves

### Commits = Save Points
Every time you change a file and commit, GitHub saves a snapshot. You can see every version of your code, forever. No work is ever lost.

### Try It

1. Go to your repo from Track 1
2. Click **Add file** → **Create new file**
3. Name it `lyrics.txt`
4. Type your favorite bar in there
5. Scroll down, write a commit message like "added my favorite lyrics"
6. Click **Commit new file**

Now click the **clock icon** (commits) at the top. You'll see your history. Every change, timestamped. That's version control.

![Commit history](assets/screenshots/02-commit-history.png)

### Branches = Alternate Versions
Want to try something without messing up what you already have?

1. Click the branch dropdown (says `main`)
2. Type a new branch name like `experiment`
3. Click **Create branch**

![Branch dropdown](assets/screenshots/02-branch-dropdown.png)

4. Make changes on this branch — your `main` branch stays untouched

When you're happy with the changes, you merge them back. That's how every developer on earth works.
