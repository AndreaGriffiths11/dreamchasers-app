# 🎵 Bonus Track: Intro to Open Source

> "The best code is built together."

`⏱️ 25 min · Bonus Track`

## What You're About To Do

Contribute to a real open source project. Open source means the code is public — anyone can see it, use it, and help improve it. Most of the software you use every day is built this way.

## Why Open Source Matters

- **Android** — open source
- **VS Code** — open source
- **Python** — open source
- **Linux** (runs most of the internet) — open source

When you contribute to open source, you're building alongside developers at Google, Microsoft, Meta, and everywhere else. Your name goes in the commit history forever.

## Your First Contribution

### The Easy Win: First Contributions
This repo is literally made for first-timers:

1. Go to [First Contributions](https://github.com/firstcontributions/first-contributions)
2. Follow the step-by-step guide
3. You'll fork the repo, add your name, and open a pull request
4. When it gets merged, you're officially an open source contributor

That's a line on your resume. For real.

### Find Projects That Want Help

Not every project is beginner-friendly. Look for these labels:
- `good first issue` — maintainers flagged this as beginner-friendly
- `help wanted` — they're actively looking for contributors
- `documentation` — you don't even need to code, just improve docs

Where to find them:
- [GitHub Explore](https://github.com/explore) — trending and recommended projects
- [Good First Issues](https://goodfirstissues.com/) — curated list
- [Up For Grabs](https://up-for-grabs.net/) — projects that want new contributors

### How to Contribute (The Flow)

```
1. Fork the repo (your own copy)
2. Create a branch (your workspace)
3. Make your changes
4. Open a pull request (send it back)
5. Respond to feedback
6. Get merged 🎉
```

That's it. Every open source contribution follows this pattern.

## The Unwritten Rules

- **Read the CONTRIBUTING.md** before you do anything. Every project has one.
- **Start small.** Fix a typo. Update a doc. Don't try to rewrite the whole thing on day one.
- **Be respectful.** Maintainers are usually volunteers. Say thanks.
- **Don't get discouraged.** Your first PR might get feedback or even rejected. That's normal. Learn from it and try again.

## Build Your Portfolio

Every contribution you make shows up on your GitHub profile as a green square. Over time, your profile tells a story: this person builds things.

Recruiters look at this. Teams look at this. It's proof you can ship.
